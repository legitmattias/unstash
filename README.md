# Unstash

Adaptive document search and organization.

Unstash makes an organization's documents searchable via natural language, with document categories discovered automatically from the actual documents — no manual tagging, no predefined taxonomies.

## Architecture

Unstash is built on a hybrid architecture that combines classical information retrieval and machine learning with targeted use of large language models:

- **Hybrid retrieval** — vector similarity search (pgvectorscale) and BM25 keyword search (ParadeDB pg_search), combined via Reciprocal Rank Fusion, both in PostgreSQL
- **Automatic categorization** — BERTopic (UMAP + HDBSCAN + c-TF-IDF) discovers document types per organization, with cluster labels generated by an LLM once per cluster
- **Classical classification** — calibrated logistic regression on embedding and TF-IDF features for fast, cheap ongoing classification
- **Universal metadata** — dates, people, organizations, and amounts extracted via NER and regex independently of document type
- **Multilingual support** — Swedish and English from day one, with Swedish as the priority for quality tuning
- **Surgical LLM usage** — LLMs are used for cluster labeling, rare classification fallbacks, and (optionally) answer synthesis, not per-document processing

## Stack

| Layer | Technology |
|---|---|
| Frontend | SvelteKit + TypeScript, Open Props, Bits UI |
| Backend | FastAPI + Python 3.12+ |
| Database | PostgreSQL 17 + pgvector + pgvectorscale + ParadeDB pg_search |
| Parsing | Docling + LibreOffice + EasyOCR |
| Embeddings | Jina AI v4 |
| Clustering | BERTopic |
| Classification | scikit-learn (calibrated logistic regression) |
| NER | KB-BERT (Swedish) + GLiNER (zero-shot) |
| Reranking | BGE-reranker-v2-m3 |
| LLM | Mistral Large 2 |
| Job queue | Taskiq + Redis |
| Infrastructure | Docker Compose, Caddy |

See `docs/adr/0001-initial-stack-choices.md` for rationale.
